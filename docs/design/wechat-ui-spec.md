# 微信小程序 UI 规格

- 状态：implemented
- 范围：`微信小程序` 登录、首页、加入会议、密码确认、入会预览、会中壳层
- 对应预览：
  - `docs/design/wechat-auth-preview.html`
  - `docs/design/wechat-room-preview.html`

## 1. 设计原则

- 与 `PC / H5` 保持同一套品牌与视觉语言：
  - 黑色舞台背景
  - `meeting` 字标
  - 字标下方椭圆聚光灯
  - 蓝色主操作按钮
  - 玻璃感深色面板
- 与 `H5` 同源，但不直接照搬浏览器交互：
  - 保留小程序导航栏和页面切换语义
  - 不依赖 `hover`
  - 弹层优先使用底部抽屉或右侧面板

## 2. 页面清单

- 登录页
- 首页
- 加入会议页
- 密码确认抽屉
- 入会预览页
- 会中壳层页

## 3. 页面规格

### 登录页

- 顶部固定元素：
  - 小程序导航栏
  - 中间标题 `meeting`
- 主内容顺序：
  - `meeting` 字标
  - 椭圆聚光灯
  - 副文案
  - 品牌标签
  - 微信快捷登录按钮
  - 状态提示卡

不可擅自变更：

- 微信快捷登录必须是主操作。
- `meeting` 字标和聚光灯必须保留。

### 首页

- 结构顺序：
  - 小程序导航栏
  - 账号信息卡
  - `加入会议` 主卡
  - `最近会议` 辅助卡

不可擅自变更：

- 首页主操作必须是 `加入会议`。
- 不得把所有入口堆成和浏览器端一样的复杂首页。

### 加入会议页

- 字段顺序：
  - `会议号`
  - `昵称`
  - 摄像头申请开关
  - 麦克风申请开关
- 底部按钮顺序：
  - `验证会议号`
  - `返回首页`

### 密码确认

- 形态：
  - 手机：底部抽屉
  - Pad：居中或偏底部浮层
- 信息顺序：
  - 已确认会议号
  - 标题 `输入会议密码`
  - 密码输入框
  - `加入并进入预览`
  - `返回修改`

### 入会预览页

- 结构顺序：
  - 本地预览舞台
  - 当前昵称与媒体状态
  - 摄像头开关
  - 麦克风开关
  - 入会前检查卡
  - `进入会议`
  - `返回修改`

与 H5 的关系：

- 逻辑保持一致
- 容器改成更像小程序的卡片和页面组织

### 会中壳层页

- 顶栏顺序：
  - 返回
  - 标题 `会议中`
  - 会议号
  - 更多按钮
- 主舞台两种状态：
  - 当前阶段的会中壳层
  - 未来音视频态占位
- 底部工具栏顺序：
  - `成员`
  - `聊天`
  - `更多`
  - `离开`

## 4. 面板规则

### 手机

- 成员：底部抽屉
- 聊天：底部抽屉
- 更多：底部抽屉

### Pad

- 成员：右侧面板
- 聊天：右侧面板
- 更多：右侧面板

## 5. 当前能力边界

- 第一阶段优先支持：
  - 微信快捷登录
  - 账号信息
  - 会议查询与加入
  - 会议密码确认
  - 入会预览
  - 基础会中壳层
- 当前不要求：
  - 浏览器级完整视频舞台
  - 白板
  - 本地录制
  - 与浏览器端完全一致的 RTC 表现

## 6. 一致性要求

- 小程序端不得额外引入另一套品牌色。
- 图标线条、圆角尺度、按钮层级要和 `H5 / PC` 保持同源。
- 文案口径必须继续使用：
  - 公开 `9` 位会议号
  - 入会预览
  - 聊天 / 成员 / 更多 / 离开

## 7. 状态要求

- 登录中、查会中、入会中都必须在当前卡片内反馈。
- 无音视频能力时，会中壳层不能出现空白黑屏。
- 错误提示必须就地展示，不跳到额外错误页。
