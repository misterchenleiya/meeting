# H5 UI 规格

- 状态：implemented
- 范围：`H5` 浏览器端从登录到进入会议首屏
- 对应预览：
  - `docs/design/h5-auth-preview.html`
  - `docs/design/h5-room-preview.html`

## 1. 目标

- 为手机浏览器与 `iPad` 这类小屏设备提供独立的 `H5` 设计稿。
- 视觉风格必须与 `PC` 浏览器端保持一致：同一套深色背景、聚光灯品牌区、蓝色主强调色、玻璃感面板、图标语义和控件层级。
- `H5` 不是把 `PC` 页面硬压缩，而是基于同一视觉系统重排信息密度、触控优先级和面板形态。

## 2. 设备分档

### 手机主档

- 基准宽度：`390px` 左右竖屏。
- 交互原则：单列、拇指优先、底部操作区优先。
- 面板策略：`sheet` / 底部抽屉优先。

### iPad / 小屏平板档

- 基准宽度：`820px` 左右竖屏。
- 交互原则：保留 `H5` 壳层，但允许局部双列和侧边浮层。
- 面板策略：聊天、成员、邀请、会议工具可切换为右侧浮层，不强制底部抽屉。

## 3. 登录到入会流程

### 页面 1：登录

- 标题顺序：
  - `meeting`
  - 副文案
  - 能力标签
- 登录方式顺序：
  - `验证码登录`
  - `密码登录`
- 输入顺序：
  - 邮箱
  - 验证码
  - 密码
- 主操作顺序：
  - `登录`
  - `微信快捷登录`
- 次级入口顺序：
  - `忘记密码`
  - `加入会议`

不可擅自变更：

- `meeting` 字标必须保留在顶部品牌区。
- 主按钮必须是蓝色强调按钮。
- `加入会议` 不允许藏入更多菜单。

### 页面 2：登录后入口

- 卡片数量：`2`
- 卡片顺序：
  - `快速会议`
  - `预定会议`
- 辅助入口：
  - `已有会议号，直接加入`
- 最近会议摘要：
  - 会议号
  - 主题
  - 时间
  - 是否需要密码

手机与 iPad 差异：

- 手机：两张能力卡纵向堆叠。
- iPad：两张能力卡允许双列并排。

### 页面 3：预定会议

- 表单字段顺序：
  - `会议主题`
  - `会议时间`
  - `时区`
  - `会议密码（可选）`
  - `备注`
- 底部按钮顺序：
  - `预定会议`
  - `返回`

手机与 iPad 差异：

- 手机：全部单列。
- iPad：前四个字段允许双列，备注独占整行。

### 页面 4：加入会议

- 字段顺序：
  - `会议号`
  - `昵称`
- 媒体申请区顺序：
  - `入会后申请摄像头`
  - `入会后申请麦克风`
- 快捷能力：
  - `扫码加入`
- 底部按钮顺序：
  - `验证会议号`
  - `返回登录`

不可擅自变更：

- 会议号仍然使用公开 `9` 位数字并按 `3-3-3` 显示。
- 流程仍然是“先验证会议号，再决定是否弹出密码确认”。

### 页面 5：密码确认

- 手机端形态：底部抽屉。
- iPad 端形态：居中悬浮卡片。
- 信息顺序：
  - 已确认会议号
  - 标题 `输入会议密码`
  - 密码输入框
  - `加入并进入预览`
  - `返回修改`

### 页面 6：入会预览

- 目标：补齐进入会议前的本地预览检查。
- 核心区块顺序：
  - 本地视频预览
  - 当前昵称与媒体状态
  - 设备标签
  - 摄像头开关
  - 麦克风开关
  - 入会前检查清单
  - `进入会议`
  - `返回修改入会信息`

手机与 iPad 差异：

- 手机：本地预览与设置区纵向排列。
- iPad：本地预览在左、设置区在右，形成 `2` 列布局。

不可擅自变更：

- 本地预览必须保留在进入会议动作之前。
- `进入会议` 仍然是蓝色主按钮。

## 4. 进入会议后的首屏

### 顶栏

- 左到右顺序：
  - 返回
  - 会议主题
  - 会议号与时长
  - 网络状态
  - 分享

### 主舞台

- 两种状态：
  - `主画面`
  - `头像墙`
- 主画面优先展示当前聚焦发言者或共享内容。
- 头像墙继续使用深色背景和无边框圆形头像体系。

### 参会者缩略条

- 位置：主舞台下方、底部工具栏上方。
- 内容：
  - 头像
  - 昵称
  - 当前媒体状态

### 底部工具栏

- 工具数量：`7`
- 工具顺序：
  - `麦克风`
  - `摄像头`
  - `共享`
  - `聊天`
  - `成员`
  - `更多`
  - `结束`

不可擅自变更：

- `结束` 必须单独保留高风险视觉状态。
- `聊天` 的未读提示只能显示红点，不显示数量。

## 5. 会中面板策略

### 手机

- 聊天：底部抽屉
- 成员：底部抽屉
- 会议工具：底部抽屉
- 邀请：底部抽屉

### iPad

- 聊天：右侧浮层
- 成员：右侧浮层
- 会议工具：右侧浮层
- 邀请：右侧浮层

## 6. 状态要求

### 加载态

- 登录发码中：按钮进入 `loading`，输入区不跳布局。
- 验证会议号中：主按钮进入 `loading`，卡片内容不闪烁。
- 入会中：密码确认层保持可见，主按钮进入 `loading`。

### 空态

- 无活动视频时，必须进入头像墙而不是空白黑屏。
- 无聊天记录时，聊天面板显示轻量空态说明。

### 错误态

- 登录失败、会议号无效、密码错误都必须就地显示，不允许跳到独立错误页。
- H5 下如果媒体能力不可用，提示要在当前页面内展示降级说明，不改变主流程结构。

## 7. 一致性规则

- 深色背景、聚光灯效果、蓝色主按钮、圆角玻璃面板、图标线条粗细必须与 `PC` 预览一致。
- `H5` 端不得额外引入全新的品牌色或卡通化视觉。
- `H5` 端不得擅自减少关键能力入口，只能重排位置和层级。
- `H5` 端的文案口径必须与当前 `PC` 流程一致，不能改成另一套产品语义。
