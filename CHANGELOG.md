# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Added

- 初始化 `Meeting` 项目骨架，明确 P2P 优先、TURN 兜底、会议数据会后清理的架构方向。
- 新增架构 ADR、后端服务入口、SQLite 审计/偏好存储和前端基础会议控制台骨架。
- 新增基础 WSS 信令通道，支持房间欢迎消息、在线状态广播、聊天消息、能力申请/授权和 SDP/ICE 消息转发。
- 新增前端 1v1 WebRTC P2P 建链能力，支持本地媒体采集、远端流渲染和基于信令的 SDP/ICE 交换。
- 新增前端屏幕共享与本地录制缓存流程，支持浏览器内存缓存、下载保存和手动丢弃。
- 新增助理授权、白板协作、就位确认和会中临时聊天/纪要运行态，支持会内查看并在会议结束后自动清理。
- 新增基于 `RTCPeerConnection.getStats()` 的基础审计统计上报与前端摘要展示，向后端周期性上报延迟、丢包、帧率和码率。
- 新增临时会议纪要查询接口 `GET /api/meetings/{meetingID}/minutes`，用于读取会中聊天、白板和就位确认快照。
- 新增昵称修改接口和前端入口，修改昵称后会自动写入临时聊天记录与会议纪要。
- 新增参会者离开会议入口和主持人保护规则，主持人在仍有其他参会者时必须显式结束会议，避免房间进入无主持人状态。
- 新增临时纪要本地导出入口，支持把会中纪要、聊天、白板数量和就位确认摘要导出为文本文件。
- 新增 `docs/design/` 设计资产目录，包含可复用的 HTML/CSS 示意页和桌面端、移动端实际渲染图。
- 新增 `Makefile`，统一提供 `build`、`run-backend`、`run-frontend` 和 `clean` 命令。
- 新增 `docs/issues/README.md`，整理最初需求与当前实现之间的功能差距和遗留问题清单。
- 新增根目录 `.gitignore`，忽略 Go/Vite 构建产物、依赖目录、日志、SQLite、本地系统文件和 `docs/design` 渲染图片。
- 新增 `README.zh-CN.md`，提供简体中文项目说明。
- 新增 `docs/design/meeting-entry-preview.html` 和 `docs/design/meeting-entry-preview.css`，补齐创建 / 加入会议状态的可复用设计源码。
- 新增 `docs/design/meeting-auth-preview.html`、`docs/design/meeting-auth-preview.css`、`docs/design/meeting-room-preview.html` 和 `docs/design/meeting-room-preview.css`，用于预览下一版黑色风格的登录 / 入会 / 会中主舞台设计。
- 新增 `docs/design/20260325-product-ui-rollout.md`，记录新版产品化前端 UI 的落地范围、兼容实现、风险与回滚思路。
- 新增会议信息分享二维码和加入会议扫码按钮，支持通过二维码快速回填会议号和密码，提升局域网联调效率。
- 新增前端结构化调试日志：浏览器控制台输出 JSON 日志，并在会中页提供“复制前端日志”入口，便于排查创建会议、入会、信令、结束会议和扫码问题。
- 新增前端日志接收接口 `POST /api/client-logs`，用于接收浏览器侧关键 JSON 日志并写入现有后端日志体系。

### Changed

- 固化权限模型：`participant` 默认仅有文字聊天权限，其他能力需主持人授权。
- 固化录制策略：录屏/录音默认本地录制，不上传服务器。
- 会中聊天改为服务端临时保存，断线重连后仍可通过房间快照恢复，会议结束后统一删除。
- 会议控制台补充了昵称编辑、离会和纪要导出操作，减少仅有后端能力但前端不可用的缺口。
- 重写 README，补齐模块职责表、需求实现状态矩阵、关键接口说明和设计资源索引。
- 前端构建输出和可控缓存目录统一收敛到 `build/`，便于集中管理构建产物和临时文件。
- README 新增 Issue 清单入口，便于后续跟踪待开发功能和技术债务。
- `README.md` 调整为英文默认文档，并补充中英文切换入口，适配面向全球的开源仓库常见做法。
- `docs/design` 目录约定调整为：HTML/CSS 预览源码和渲染脚本入库，示意图图片按本地产物处理并加入忽略规则。
- `docs/design/render-previews.sh` 改为基于脚本目录解析路径，避免依赖开发者本机绝对路径。
- `docs/design/meeting-console-preview.html` 和样式文件补齐了就位确认、本地录制、事件流等已实现的会中区域，和当前前端页面结构保持一致。
- `docs/design/README.md` 和 `docs/design/render-previews.sh` 补充下一版产品预览入口，支持单独渲染登录 / 入会页和会中舞台页的桌面端、移动端图片。
- `docs/design/meeting-auth-preview.html` 支持按视图状态定向渲染，`docs/design/render-previews.sh` 新增主持人视角的登录、创建会议和进入会议三张独立示意图输出。
- 后端默认 HTTP 监听端口改为 `5180`，前端本地开发代理同步切换到 `127.0.0.1:5180`，避免与本机已占用的 `8080` 端口冲突。
- 前端开发服务器默认端口改为 `5188`，避免与本机已占用的 `5173` 端口冲突。
- 前端主界面按最新 `docs/design` 重构为黑色风格产品化 UI，补齐登录壳层、登录后入口、预定会议 / 快速会议、两段式加入会议、会中主舞台、右侧成员 / 聊天抽屉和邀请 / 录制申请弹窗。
- 会中布局从旧版综合控制台切换为“黑色主舞台 + 右侧缩略窗 + 底部工具栏 + 下方辅助面板”的结构，并保留白板、就位确认、临时纪要、审计摘要等既有能力。
- 主持人登录页切换为测试模式，预填 `meeting@07c2.com.cn / helloworld`，并允许任意非空邮箱和密码直接登录，便于本地联调创建会议。
- `make build` 的产物路径调整为：后端输出到 `build/backend/meeting`，前端输出到 `build/frontend/`，便于区分前后端构建结果。
- 前端结构化日志改为仅保留浏览器控制台输出和当前页面内存中的最近日志，不再写入浏览器 `sessionStorage`；`warn`/`error` 和关键 `info` 事件会按批次上报到后端日志。
- 前端开发态 `WebSocket` 信令改为直接连接后端 `:5180`，不再经过 Vite 的 `/ws` 代理；主舞台头像墙仅展示自己和当前在线的参会者，减少断线残留对会中视图的干扰。

### Fixed

- 后端日志输出统一为包含 `level`、`time`、`message` 的 JSON 格式，并将 `level` 调整为小写，满足项目全局日志规范。
- 后端日志文件补齐 24 小时轮转和最近 3 天保留策略，避免长时间运行时单文件持续增长且依赖重启才清理旧日志。
- 后端启动早期的日志初始化失败场景改为使用同一套 JSON 日志格式输出到标准错误，避免绕过项目日志规范。
- 修复登录页空闲时前端会因 `useEffectEvent` 被错误写入 `useEffect` 依赖而反复执行 RTC 清理，导致浏览器控制台持续刷出 `Maximum update depth exceeded`。
- 修复会中基础审计上报和入会扫码的前端 effect 依赖误用，避免主持人开会后审计日志以毫秒级频率刷屏，并消除同类扫码生命周期隐患。
- 修复从首页进入快速会议后页面可能保留原有滚动位置的问题，避免会中首屏落在黑色舞台中段时看起来像整页空白，进入和退出会议时会自动回到顶部。
- 修复主持人首次进入快速会议时信令连接可能在几十毫秒内被过期回调误关的问题，避免会中首屏出现整页黑场且必须手动刷新才能恢复。
- 修复部分浏览器环境缺少 `crypto.randomUUID()` 时快速会议首进流程会在本地事件写入阶段直接报错，避免主持人进入会议后出现整页黑场并只能靠刷新恢复。
- 修复会中点击摄像头、麦克风和屏幕共享按钮时对 `navigator.mediaDevices` 的兼容性判断缺失问题；在不支持或非安全上下文环境下不再直接抛前端异常，而是给出明确的浏览器与访问方式提示。
- 会议密码改为真正可选：快速会议和未设置密码的预约会议允许参会者仅凭会议号直接加入，不再强制弹出密码输入。
- 主持人结束会议前新增二次确认，结束后可先导出当前会议纪要，再返回预约会议或快速会议入口，降低误触成本。
- 前端会中刷新后会自动恢复当前会话并重连信令，不再直接回到登录页。
- 结束会议确认框改为按整个浏览器视口居中显示，不再跟随视频舞台区域定位，避免误以为点击结束会议没有反应。
- 修复主持人结束会议时的请求体字段不匹配问题，确认结束会议后不再被后端以 `invalid request body` 拒绝。
- 加入会议扫码在不支持 `getUserMedia` 的移动端环境下改为给出明确提示，并提供拍照 / 选图识别二维码的降级路径；图片识别会先缩放再解码，避免手机原图过大时看起来“没有反应”。
- 修复开发联调时 `WSS` 信令频繁断开的问题：前端改为直连后端信令地址，后端在参会者断开连接后会延迟清理并广播离会，避免关闭页面后人数持续累加。
